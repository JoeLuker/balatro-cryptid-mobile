# Balatro Cryptid Mobile

**Two twin projects, one repo:**

1. **The LÖVE Cryptid build** (this README; everything outside `rebuild/`) —
   packages vanilla `Balatro.love` + the Cryptid / Talisman / Steamodded mods into a
   signed Android APK via a pinned, reproducible Nix pipeline. Mod code is **baked
   into the APK** — nothing is live-patched or pushed to the device's save dir.
2. **The Kotlin rebuild** (`rebuild/`; [its own README](rebuild/README.md)) — a
   from-scratch native Kotlin / Jetpack Compose reimplementation of the game,
   verified against the LÖVE build. Its own Gradle project: `cd rebuild`.

The twins meet only at the verification seams: `test/kt-oracle.sh` (scoring parity),
`tools/uiref/` (extracted vanilla UI geometry), and the reference trees below.

## Reference trees — which source to read

| Tree | What it is | Use it for |
|---|---|---|
| `src/vanilla/` | pure vanilla source (readable extraction of `src/Balatro.love`; gitignored) | rebuild fidelity questions |
| `vendor/dump/` | committed lovely-merged tree (vanilla + mods) — what the APK actually ships | LÖVE-build patch questions |

Recreate the extraction any time: `unzip -oq src/Balatro.love -d src/vanilla`.
`src/Balatro.love` itself is pinned via `nix/sources.json` (`requireFile` — © LocalThunk,
supply locally; never committed or redistributed).

# The LÖVE Cryptid build

## Quick start

```bash
just build            # nix build → build/game + signed build/apk (pinned, reproducible)
just deploy           # install the APK on the connected phone (APK only — see Deploy)
just test             # the full local gate (no phone needed) — run before any deploy
```

All inputs are pinned in `nix/sources.json` (no network `latest`). The build is a Nix
derivation: pinned pristine sources → patch series → `game.love` → signed APK.
See `MIGRATION.md` for how this architecture came to be.

## How it works

1. **Pins** — `nix/sources.json` locks every input (game, base APK, each mod) by hash;
   `nix/update-sources.sh` re-pins.
2. **Assemble** — `nix/balatro-cryptid.nix` stages the pristine game + mods
   (`gameLoveBase`), with the lovely-merged dump vendored at `vendor/dump/`
   (regenerate from pins: `nix/regen-dump.sh`, boots love+lovely under Xvfb).
3. **Patch** — `overlay/patches/` (mechanical, generated by `nix/gen-patches.sh` from
   `scripts/build.sh`'s `apply_*` + `scripts/patch_main_lua.py`) and
   `overlay/patches/manual/` (hand-maintained, numeric order) apply hard-fail via the
   `series` file. `overlay/game/` files are copied over the tree afterwards;
   `overlay/config/` bakes the mod configs in.
4. **Package** — `game.love` is embedded in the base APK; `nix/sign.sh` signs with `keys/`.

## Deploy

`just deploy` **installs the APK only**. Mod code lives solely in the APK's embedded
`game.love`; the deploy step also cleans any stale code files out of the device's
`files/save/Mods/`, preserving per-mod `config.lua` settings (LÖVE's filesystem overlay
reads a preserved config while the absent code file falls through to the APK copy).
Never push mod code to the save dir — a second writable copy silently shadows the APK.

## Local testing (no phone required)

Three layers, fastest first — `just test` runs the full gate:

1. **Gesture suite** (`just test-controller`, <1s): drives the real built
   `engine/controller.lua` with scripted tap/hold/drag timelines under pure LuaJIT.
2. **Desktop smoke** (`just smoke`, ~1 min): boots `build/game` under Xvfb with
   `love.system.getOS()` spoofed to `'Android'`; asserts boot-to-menu + screenshot.
3. **Emulator** (`just emu-test`, ~5–10 min): installs the signed APK in a headless
   Android emulator and polls for a menu frame vs the crash-screen signature.

What still needs a real phone: Mali-GPU shader behaviour, touch feel, performance.

## Project structure

```
balatro-cryptid-mobile/
├── flake.nix / justfile      # `just build|deploy|test` (nix under the hood)
├── nix/                      # pins (sources.json), build derivation, gen-patches, regen-dump, sign
├── overlay/
│   ├── patches/  *.patch + series + manual/   # our diffs vs pristine (hard-fail)
│   ├── game/                 # whole files we own (conf.lua)
│   └── config/               # baked-in mod config (Cryptid/, Steamodded/)
├── vendor/dump/              # committed lovely-merged game lua (regen from pins)
├── patches/                  # runtime modules loaded from main.lua (telemetry, perf, …)
├── scripts/build.sh          # deploy/logs/clean + the apply_* source gen-patches reads
├── src/                      # Balatro.love (supply locally) + vanilla/ extraction
├── test/                     # LÖVE-build harnesses + the kt-oracle parity seam
├── tools/uiref/              # UI-geometry extraction oracle (shared with rebuild)
├── rebuild/                  # ← the Kotlin twin (own README, own Gradle project)
└── build/  (gitignored)      # build/game, build/apk
```

## Mods

Mods are **pinned inputs**, not files you drop in:

1. Pin the mod in `nix/sources.json` (`nix/update-sources.sh` re-pins from upstream).
2. Wire it into the mod list in `nix/balatro-cryptid.nix`.
3. If the mod lovely-patches game code (`lovely/*.toml`), regenerate the vendored dump
   from the pins: `nix/regen-dump.sh` (needs `tools/lovely-src`, built once locally).
4. `just build` — patches apply hard-fail, so drift is loud, not silent.

**Config**: baked configs live in `overlay/config/<Mod>/config.lua` — edit, rebuild,
deploy. Settings changed on-device persist independently (`config.lua` files under the
save dir are preserved across deploys).

## Runtime patch modules (`patches/`)

Loaded from `main.lua`'s tail (injected by `scripts/patch_main_lua.py`):
`android-nativefs` (FFI-free nativefs for Android), `android-telemetry` (the OBS suite —
see `docs/OBSERVABILITY.md`), `trigger-collapse` (event-cascade collapsing),
`idle-joker-perf` (align_cards sort gate + card-limit cache), `lazy-shader`
(deferred shader binds), `emulator-smoke-check` (dormant off-emulator).
`crt-shader-fix.patch` fixes the Tensor-GPU float-overflow black oval.

## Troubleshooting

- **App crashes on boot** → `just logs`; look for `LOAD_FAILED`, SMODS, or NATIVEFS lines.
- **Black oval on screen** → CRT shader unpatched; rebuild.
- **A fix "doesn't take effect on device"** → verify the APK actually reinstalled and
  that `files/save/Mods/` holds nothing but `config.lua` files — never re-add a save-dir
  code push to "fix" this.
