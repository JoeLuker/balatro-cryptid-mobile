## 1. Kotlin rebuild — correctness (all hand-verified or double-agent-verified)

### Boss-blind selection is non-vanilla three ways — `game/Blinds.kt`
- **HIGH** `Blinds.kt:104` — showdown pool gated on hardcoded `ante >= 10`. Vanilla
  `get_new_boss()` uses `ante % win_ante == 0` (default win_ante **8**): the Ante-8
  finale boss never appears, and post-win antes 9–15 are wrong in both directions.
- **HIGH** `Blinds.kt:107` — `pool()` ignores each boss's vanilla min-ante unlock gate
  (late-game bosses can appear at Ante 1).
- **MED** `Blinds.kt:109` — no `min_use` anti-repeat weighting (vanilla cycles the full
  pool before repeats).
- **LOW** `Blinds.kt:38` — no per-boss `dollars` field; showdown bosses pay $8 in
  vanilla, callers hardcode $5.

### Save subsystem loses state — `save/`
- **HIGH** `RunSnapshot.kt:48` — `consumableSlotsBonus` (Crystal Ball voucher,
  Magic/Nebula decks; `RunScreen.kt:883`) is never persisted: save/reload silently
  shrinks consumable slots back to 2.
- **MED** `RunScreen.kt:822` — run-lifetime counters (totalHandsPlayed, runHighScore,
  per-hand-type play counts, cumulative chips/cards) not in the snapshot → joker
  trigger cadences desync after reload.
- **MED** `SaveIo.kt:12` — save writes are not atomic (no write-temp-then-rename);
  process death mid-write corrupts the save, and recovery discards the run.
- **LOW** `RunSnapshot.kt:87` — `schemaVersion` is written but never read; the
  migration mechanism its comment describes doesn't exist.

### UI state-management bugs — `ui/RunScreen.kt`
- **HIGH** `:3289` (and `:3829` shop variant) — joker selection is a **raw list
  index** (`selJoker: Int?`), only bounds-checked. Index-agnostic removals in the same
  phase (`owned.removeAll` for j_ramen/j_selzer/j_cry_blacklist self-destructs during
  scoring; j_cry_starfruit on shop reroll) silently retarget the inspect/sell panel to
  a *different* joker. Reachable by ordinary play.
- **HIGH** `:1918` — `sell()` ignores the return of `owned.remove(o)` and pays the
  refund + runs every per-sell hook regardless; `redeemVoucher()` (`:2043`) re-applies
  its one-shot effect on every call and nulls the offer only at the end. Unlike
  `cashOut()`/`useTarot()`/`pickPackItem()` in the same file, neither guards
  re-invocation → fast double-tap duplicates money/effects.
- **MED** `:1852` — the `buy*()` family never re-validates the offer is still present
  in the shop before mutating state.

### Layout fidelity — `ui/`
- **HIGH** `RunScreen.kt:3136` — play-area `alignCards(tempLimit = max(scoreCards.size, 1))`
  overrides the already-correct default (`cardLimit = 5`). Vanilla spaces played cards
  against `max(#cards, 5)` (cardarea.lua:319 + game.lua's `card_limit = 5`), so every
  <5-card hand fans wrong. Deleting the override fixes it.
- **MED** `RunScreen.kt:3135` — hand fan-spacing hardcodes `tempLimit = 8` instead of
  the live, voucher/boss-mutable hand size.
- **MED** `HudSpec.kt:156` — extracted-tree parser defaults missing `config.align` to
  `'cm'` (vanilla default is *no* alignment), and B-node children are attached but
  never laid out. Inert against the currently-committed trees; will bite the next
  ported screen and contradicts the file's "nothing can be silently altered" claim.

### Audio
- **MED** `audio/MusicManager.kt:19` — `start()` can call `MediaPlayer.start()` while
  still async-Preparing → IllegalStateException race.
