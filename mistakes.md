### LÖVE shader highp injection fails
adding 'precision highp float;' after LÖVE's preamble breaks compilation on Mali with "'highp': overloaded functions must have the same parameter precision qualifiers" — LÖVE's built-ins (Texel/samplers) use the default precision and mixing highp user code with them is rejected. Global precision override is NOT a viable mobile-compat fix for LÖVE shaders. To get fp32 you'd need consistent per-variable highp including function params, or rewrite the math to stay in fp16 range.
